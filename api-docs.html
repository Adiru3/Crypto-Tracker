‚öôÔ∏è How It Works

The API uses CoinGecko as the primary data source and implements smart caching to ensure fast responses and avoid rate limits. Here's the architecture: 

Your App ‚Üí Crypto Tracker API ‚Üí Cache Layer ‚Üí CoinGecko ‚Üí Real Data

Response time: <100ms (cached) | <500ms (fresh data)
üìå Note: All data is cached for 5 minutes to ensure optimal performance. This means you can make frequent requests without worrying about rate limits!

üîå API Endpoints

GET /api/coins

Get a list of all supported cryptocurrencies with current prices and market data.

Example Request:

fetch('https://your-api-domain.com/api/coins')
  .then(res => res.json())
  .then(data => console.log(data));

Example Response:

[
  {
    "id": "bitcoin",
    "symbol": "btc",
    "name": "Bitcoin",
    "image": "https://...",
    "current_price": 45000,
    "market_cap": 880000000000,
    "market_cap_rank": 1,
    "price_change_percentage_24h": 2.5,
    "total_volume": 25000000000
  },
  ...
]
GET /api/coins/:id

Get detailed information about a specific cryptocurrency.

Parameters:

Parameter Type Description
id string Coin ID (e.g., 'bitcoin', 'ethereum')

Example:

GET /api/coins/bitcoin
GET /api/coins/:id/chart

Get historical price data for charts.

Parameters:

Parameter Type Description
id string Coin ID
days number Number of days (default: 30)

Example:

GET /api/coins/bitcoin/chart?days=7
